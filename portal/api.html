API Reference

Endpoints exposed by this device. Live device info: /api

Device Info

MethodPathBody / ResponseDescription
GET/api{vendor, apiVersion, firmwareVersion, nodeName, ledPin, ledClkPin, ledCount, ledType, lampType, lampForm, physicalW/H, logicalW/H, pixelGroupW/H, rotation, origin, serpentine, serpentineAxis}All device metadata.
GET/api/state{on, color:[r,g,b], mode, brightness, current}Current lamp state. color is the user-set baseColor (last value sent through /api/color) — stable across modes; clients sync color pickers from this. current is the active JS script when mode == "js".
GET/api/base_color{color: [r, g, b]}Just the baseColor, without the rest of the state. Same value as state.color; useful when the client only needs the color and wants a smaller payload.
GET/api/storage{total, used, free, files}SPIFFS storage usage in bytes + script file count.

Scripts

MethodPathBody / ResponseDescription
GET/api/js{scripts:[...], playing:<name>|null}List scripts and currently playing one.
GET/api/js/{name}raw JS sourceRead a single script.
PUT/api/js/{name}JS source bodySave script. Validated + eval'd once before write.
DELETE/api/js/{name}—Delete a script (and its params sidecar).
GET/api/js/{name}/params{items: [{name, min, max, default, value, description}, ...]}Tunable parameters for a script. Schema is parsed from // @param magic comments in the script source; persisted overrides are merged in. Empty items if the script declares none.
PUT/api/js/{name}/params{"<param>": value, ...}Update one or more params. Values are clamped to [min, max], persisted to a sidecar file, and applied live to the player without restarting the animation if this is the currently-playing script. Returns the post-update schema.
POST/api/play{"file":"<name>","fps":N}Run a script at the given FPS.
POST/api/play/next{playing:<name>}Step to the next script alphabetically and play it.
POST/api/play/prev{playing:<name>}Step to the previous script alphabetically and play it.
GET/api/play/current{current:<name>|null, playing:<name>|null}current is the persisted selection (NVS, survives reboot); playing is what the player has loaded right now.
POST/api/stop—Stop JS playback.

Magic-comment parameters. Add // @param NAME MIN..MAX = DEFAULT DESCRIPTION lines anywhere in a script to expose tunable knobs. Numeric only; up to 8 per script. Example:
// @param density 0..1 = 0.75 Fraction of LEDs animating concurrently
The script receives a params object as the 5th argument to render(frame, w, h, baseColor, params); read params.density etc.

WebSocket Streaming

Connect to ws://<ip>/ws for continuous LED streaming. Frames are sent as binary messages.

BytesFieldDescription
[0]command0x01 color frame, 0x02 power, 0x03 clear.
[1..2]led count (uint16 BE)0x01 only.
[3..]RGB bytes0x01 only — 3 bytes per LED (R,G,B). Logical raster; device handles pixel grouping + rotation + serpentine.
[0x02,b]powerb=0 off, b=1 on.
[0x03]clearAll LEDs off.
